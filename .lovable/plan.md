

# Корпоративный сайт JIMON Group — реализация по договору

Согласно договору AL-JG-2026/04-01: многостраничный корпоративный сайт **JIMON TORANGY DAILY NECESSITES ЖШС** на базе референса **jinmujituan.cn**, адаптивный, с CMS и локализацией на 5 языков (RU / KZ / UZ / EN / KG).

---

## 1. Полная замена текущего проекта

Текущий код (трекер привычек DailyMint) **удаляется полностью**. Старые маршруты (`app.tsx`, `insights.tsx`, `settings.tsx`) и компоненты (`HabitCard`, `BottomNav`, `OnboardingFlow` и т.д.) больше не нужны.

Удаляются: все файлы под `src/components/` (кроме `ui/`), `src/lib/habits*`, `src/hooks/use-theme.ts`, `src/components/CalendarHeatmap.tsx` и пр. Старые таблицы БД (`habits`, `habit_logs`) сохраняются в БД, но не используются (можно очистить отдельной миграцией позже).

---

## 2. Структура страниц (маршруты)

```
/                       — Главная (hero, миссия, направления, новости)
/about                  — О компании (история, руководство, ценности)
/brands                 — Наши бренды (международное видение, продуктовые линейки)
/products               — Продукция (категории + сетка карточек)
/news                   — Новости (список)
/news/$slug             — Детальная новость
/contact                — Контакты (адрес, форма обратной связи)
/login                  — Вход в админку
/admin                  — Админ-панель (защищена)
/admin/news             — CRUD новостей
/admin/brands           — CRUD брендов
/admin/products         — CRUD продуктов
/admin/translations     — Редактор переводов UI-строк
```

Все публичные маршруты используют общий `Header` (логотип, меню, переключатель языка) и `Footer` (контакты, реквизиты, соцсети).

## 3. Локализация (5 языков)

- Библиотека `i18next` + `react-i18next`. Переключатель в шапке (RU / KZ / UZ / EN / KG).
- Язык по умолчанию — **русский**, выбор сохраняется в `localStorage`.
- **Статичные UI-строки** (меню, кнопки, лейблы) — в JSON-словарях `src/locales/{lang}/common.json`. Переводы выполню через Lovable AI (Gemini) при инициализации проекта, исходник на русском.
- **Динамический контент** (новости, бренды, продукты) хранится в БД с колонками `name_ru`, `name_kz`, `name_uz`, `name_en`, `name_kg` и аналогично для описаний. Если перевод пуст — fallback на русский.

## 4. Дизайн-система (на основе референса)

Референс — корпоративный «зелёно-золотой» стиль. Делаю свежую адаптацию для CA/СНГ-рынка:

- **Палитра**: основной зелёный `#1B5E3A` (доверие, природа), акцент золотой `#C9A24A`, фоны кремовый `#F8F5EE` и тёмно-зелёный `#0E2A1C`.
- **Шрифты**: заголовки — `Manrope` (700/800), основной текст — `Inter` (400/500). Кириллица + латиница.
- **Hero**: полноэкранный баннер с градиентом + декоративный лиственный паттерн, крупный заголовок «JIMON — Восточная сила природы для всего мира».
- **Карточки направлений**: 5 блоков с иконками (Продукция / Здоровье / Глобальное видение / R&D / Социальная ответственность) — повторяют структуру референса.
- **Адаптив**: Desktop / Tablet / Mobile через Tailwind breakpoints. Мобильное меню — выдвижной drawer.
- Все `shadcn/ui` компоненты остаются; темы переопределяю в `src/styles.css`.

Иллюстрации: AI-сгенерированные баннеры (через `google/gemini-3-pro-image-preview`) в стилистике натуральной медицины + плейсхолдеры брендов/продуктов из референса.

## 6. Бэкенд (Lovable Cloud)

Новые таблицы:

- **`user_roles`** (`user_id`, `role` enum: `admin`) + security-definer функция `has_role()` — проверка админки без рекурсии RLS.
- **`news`** (id, slug, cover_url, published_at, is_published, title_ru/kz/uz/en/kg, body_ru/kz/uz/en/kg). RLS: SELECT для опубликованных — публичный; всё остальное — только админ.
- **`brands`** (id, slug, logo_url, position, name_*, description_*). RLS аналогично.
- **`products`** (id, slug, image_url, category, position, name_*, description_*). RLS аналогично.
- **`translations`** (key, ru, kz, uz, en, kg) — для UI-строк, редактируемых из админки. Публичный SELECT, INSERT/UPDATE — только админ.
- **`contact_messages`** (id, name, email, phone, message, created_at) — заявки с формы. Публичный INSERT с валидацией Zod, SELECT — только админ.
- **Storage bucket** `media` (публичный для чтения) — изображения новостей, брендов, продуктов.

Аутентификация: email/пароль + Google OAuth. После регистрации пользователь не получает `admin` автоматически — роль выдаётся вручную через SQL (или первой заведённой записи). На странице `/login` — стандартная форма.

## 7. Админ-панель `/admin`

Защищена layout-роутом `_admin.tsx` через `beforeLoad` + `has_role('admin')`. Внутри:

- Простой sidebar с разделами.
- CRUD-таблицы для новостей, брендов, продуктов с переключателем языковых вкладок (RU / KZ / UZ / EN / KG) при редактировании.
- Загрузка изображений в `media` bucket.
- Редактор переводов UI-строк (поиск + inline-редактирование).
- Просмотр заявок с формы контактов.

«Обучение представителя» из п. 2.1 договора — отдельная страница `/admin/help` с краткой инструкцией.

## 8. SEO (базовый уровень из договора)

- Уникальные `<title>` и `<meta description>` в `head()` каждого роута (через TanStack Router).
- `og:title`, `og:description`, `og:image` для главной и новостей.
- `public/robots.txt` — разрешить индексацию.
- `/sitemap.xml` — server route, генерирует список из БД (новости + статичные страницы).
- `lang` атрибут на `<html>` динамически по выбранному языку.

## 9. Технические детали

- **Стек**: TanStack Start + React 19 + Tailwind v4 + shadcn/ui (уже есть). i18next добавляется.
- **Маршрутизация**: file-based routes, защита админки через `_admin` pathless layout.
- **Роли**: отдельная таблица `user_roles` + `has_role()` SECURITY DEFINER (по правилам Lovable).
- **Миграции БД**: одна миграция с enum `app_role`, всеми таблицами, RLS-политиками, триггерами и storage bucket.
- **AI**: Lovable AI (`google/gemini-2.5-flash`) для генерации словарей переводов + `gemini-3-pro-image-preview` для hero-баннеров.
